Orion-Belt Documentation

Orion-Belt is an open-source, self-hosted Privileged Access Management (PAM) system that provides secure SSH/SCP access to infrastructure through a bastion host architecture with reverse SSH tunneling.

What is Orion-Belt?

Orion-Belt acts as a secure gateway between users and your infrastructure, providing authentication, authorization, session recording, and comprehensive audit logging for all SSH/SCP connections—without requiring any inbound firewall rules.

Why Orion-Belt?​

Traditional SSH and VPN-based access models have significant limitations:

  • Long-lived credentials with no expiration
  • No native approval workflow for temporary access
  • Limited auditability of who accessed what and when
  • Broad network access instead of granular, per-machine control
  • Open inbound ports that increase attack surface

Orion-Belt solves these problems by:​

Eliminating inbound firewall rules using reverse SSH tunnels
Enforcing fine-grained, relationship-based access control (ReBAC)
Supporting temporary, approval-based access workflows
Recording every session for audit and replay
Acting as a single, centralized access gateway

Think of it as a lightweight, self-hosted alternative to enterprise solutions like Teleport, BeyondTrust, or CyberArk—built with simplicity, auditability, and extensibility in mind.

Key Features​

Zero-Trust Architecture​

  • Reverse SSH tunneling - Agents connect outbound to the gateway
  • No inbound firewall rules required on target machines
  • Gateway acts as the single point of entry

Access Control​

  • ReBAC (Relationship-Based Access Control) with OpenFGA integration
  • Fine-grained permissions per user, per machine
  • Support for temporary access grants with expiration
  • Admin approval workflows for sensitive access

Session Recording & Audit​

  • Complete SSH session recording with input/output capture
  • Timestamp-based replay capability
  • Comprehensive audit logs for compliance
  • API access to historical sessions

Extensible Architecture​

  • Plugin system for custom integrations
  • Notification hooks (Slack, email, webhooks)
  • Database-agnostic design (PostgreSQL reference implementation)
  • REST API for programmatic access

Production-Ready​

  • Docker containerization
  • Multi-platform support (Linux, macOS)
  • Alpine Linux optimized deployments
  • Comprehensive API with 40+ endpoints

Architecture Overview​

Orion-Belt consists of three main components:

1. Gateway Server​

The central control plane that:

  • Manages all SSH/SCP proxy connections
  • Enforces authentication and authorization
  • Records all session activity
  • Exposes REST API for management
  • Handles access request workflows

2. Agent​

Lightweight daemon deployed on target machines that:

  • Establishes reverse SSH tunnel to gateway
  • Receives connections from the gateway
  • No inbound ports required
  • Auto-reconnects on failure

3. Client Tools​

Custom CLI tools that:

  • osh - Orion-Belt SSH client for interactive sessions
  • ocp - Orion-Belt SCP client for file transfers
  • Integrate with gateway API
  • Support public key authentication

How It Works​

  1. Agent Registration: Agents connect to the gateway using reverse SSH, establishing persistent tunnels
  2. User Authentication: Users authenticate to the gateway using SSH public keys
  3. Authorization Check: Gateway validates permissions using ReBAC before allowing access
  4. Session Proxy: Gateway forwards the connection through the agent to the target machine
  5. Recording: All session I/O is captured and stored for audit

Use Cases​

Enterprise Infrastructure Access​

  • Provide developers secure access to production servers
  • Enforce approval workflows for sensitive systems
  • Maintain comprehensive audit trails for compliance

Homelab Management​

  • Secure access to homelab infrastructure
  • No need to expose SSH ports publicly
  • Single point of entry for all machines

Research & Education​

  • Grant temporary access to shared resources
  • Track usage for accounting
  • Revoke access automatically after expiration

Cloud & Hybrid Environments​

  • Unified access across on-premises and cloud
  • Works behind NAT and firewalls
  • No VPN required
