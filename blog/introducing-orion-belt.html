Introducing Orion-Belt - Open Source PAM for Modern Infrastructure

January 10, 2026 · 2 min read
Mohamed Zrouga
Creator of Orion-Belt

After months of development and testing in production environments, I'm releasing Orion-Belt as an open-source Privileged Access Management system. This project started from a practical need: managing secure SSH access across multiple machines without the complexity and cost of enterprise solutions.

Why Another PAM Solution?​

Commercial PAM solutions like Teleport offer excellent features but come with licensing costs that can be prohibitive for smaller teams, homelabs, and organizations just starting their security journey. Orion-Belt provides a self-hosted alternative that covers the essential requirements:

  • Zero-trust architecture using reverse SSH tunnels
  • Fine-grained access control with relationship-based permissions
  • Complete session recording for audit and compliance
  • Temporary access workflows with approval mechanisms
  • Extensible plugin system for custom integrations

Architecture Decisions​

The design prioritizes security and simplicity. Agents on target machines initiate outbound connections to the gateway, eliminating the need for inbound firewall rules. This reverse tunnel approach means your infrastructure remains unexposed even as you provide SSH access through the gateway.

Access control uses OpenFGA for relationship-based permissions, allowing natural expressions like "alice can access machine-32 as root" or "bob can access any machine in the production environment as the deploy user."

Current Status​

Version 0.2.0 is stable and running in production. The core functionality is complete:

  • Gateway server with SSH/SCP proxy and REST API
  • Lightweight agents for Linux systems (Alpine, Debian, Ubuntu, RHEL)
  • Custom client tools (osh for SSH, ocp for SCP)
  • PostgreSQL backend for persistence
  • Session recording with playback capability

The next major release (0.3.0) will add multi-factor authentication and SSH Certificate Authority support.

Getting Started​

Installation takes about 10 minutes. The documentation covers setup for the gateway server, deploying agents, and configuring your first connection. Docker images are available, and installation scripts support multiple Linux distributions.

Check the Quick Start guide for detailed instructions.

Contributing​

The project is open for contributions. I'm particularly interested in:

  • Testing on different Linux distributions
  • Bug reports and use case feedback
  • Documentation improvements
  • Plugin development examples

Visit the GitHub repository to get involved.
